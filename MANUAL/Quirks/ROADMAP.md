# Роадмап настройки Raspberry Pi 4 с внешним SSD (Quirks Mode)

Этот документ содержит пошаговое описание решения проблемы загрузки Raspberry Pi 4 с внешнего SSD Kingston через проблемный USB-SATA адаптер.

---

## 1. Проблема: Несовместимость UAS
Многие USB 3.0 адаптеры (включая твой на чипе OWC/JMicron) имеют "кривую" поддержку протокола **UAS (USB Attached SCSI)**. На Raspberry Pi 4 это приводит к:
* Циклическим перезагрузкам при загрузке.
* Ошибкам в консоли: `reset high-speed USB device number...`.
* Полному зависанию системы при обращении к диску.

## 2. Решение: Применение "Quirks"
**Quirks** — это параметр ядра Linux, который заставляет систему игнорировать UAS и использовать стабильный драйвер `usb-storage`.

### Идентификация устройства
Через команду `lsusb` был определен ID твоего адаптера:
**`7825:a2a4`** (Other World Computing External SATA Hard Drive Adapter).

---

## 3. Пошаговая инструкция по исправлению (через Windows)

Если малинка не грузится, сделай это на обычном компьютере:
1. Подключи SSD к ПК. Windows увидит раздел объемом ~250 МБ (называется `boot` или `bootfs`). **НЕ ФОРМАТИРУЙ** другие разделы, если Windows предложит!
2. Открой файл **`cmdline.txt`** в Блокноте.
3. Добавь в **самое начало** строки следующий текст:
   `usb-storage.quirks=7825:a2a4:u`
   *(Обязательно добавь пробел после буквы `u`, чтобы отделить его от следующих команд).*
4. Сохрани файл и вставь SSD обратно в синий (USB 3.0) порт Raspberry Pi.

---

## 4. Настройка системы (OS Lite / Debian)
После успешной загрузки до экрана `login:`:

### Вход в систему
* **Логин:** `root`
* **Пароль:** `root` (или тот, который ты задавал при создании образа).

### Сброс пароля пользователя `pi`
Если ты вошел под `root`, но хочешь использовать юзера `pi` (например, для SSH):
1. Введи команду: `passwd pi`
2. Введи новый пароль дважды (символы при вводе скрыты).

### Настройка SSH (Удаленный доступ)
Чтобы управлять малинкой с компьютера через терминал:
1. Включи службу SSH:
   `systemctl enable --now ssh`
2. (Опционально) Чтобы разрешить вход под `root` по SSH:
   * Отредактируй конфиг: `nano /etc/ssh/sshd_config`
   * Найди/измени строку на: `PermitRootLogin yes`
   * Перезапусти SSH: `service ssh restart`

---

## 5. Итоговая команда для подключения с ПК
`ssh pi@192.168.100.104` (замени IP, если он изменится в будущем).

---
**Важно:** Если ты решишь переустановить систему или сменить SSD/MicroSD, тебе **снова** придется добавить `usb-storage.quirks=7825:a2a4:u` в новый файл `cmdline.txt`. Сохрани этот ID!
