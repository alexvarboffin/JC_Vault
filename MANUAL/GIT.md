  5. Точка расхождения (Common Ancestor (или «Общий предок»))
  У вашего локального кода и кода на сервере есть общий предок — это последний коммит, который был у
  вас обоих до того, как пути разошлись.

  6. Как выглядит Merge (обычный git pull):
  Git видит, что от общего предка история пошла в две стороны:
   * Путь А (Сервер): 8 новых коммитов.
   * Путь Б (Вы): 2 ваших коммита с SIP.

  Когда вы делаете merge, Git создает Merge-коммит, у которого два «родителя». Он как бы говорит
  истории: «Возьми всё из Пути А и всё из Пути Б и склей в этой точке».

  На сервере после пуша это будет выглядеть так:

   1           (Ваш 1)---(Ваш 2)
   2          /                 \
   3 ---(Общий)---(Серв 1)---...---(Серв 8)---(MERGE КОММИТ)  <-- HEAD
  Визуально это выглядит как «петля» или «развилка», которая потом сошлась. Все коммиты (8 + 2 + 1)
  становятся частью истории main.
